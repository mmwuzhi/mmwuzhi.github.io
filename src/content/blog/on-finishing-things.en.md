I have a lot of unfinished projects.

Not just code — half-read books, articles with only an opening paragraph, apps installed and uninstalled. This has bothered me for a long time. But lately I've started to see it more clearly: **finishing** isn't the goal in itself, but it's the only way to make something exist in the world.

## Why we never finish

Starting something is easy, because at that point everything is still possible. The middle is the hardest, because you already know it won't be as good as you first imagined — but you still can't see the end.

> Finishing is a kind of compromise with reality, and a kind of accountability to yourself.

## A small strategy

I've started asking myself before beginning anything: **"What would I be okay with this becoming?"**

Not the ideal version — just the version I could hand over, the version someone else could see. The more concrete that expectation, the more likely I actually finish.

This post got written, probably because I decided from the start it didn't need to be very good.
